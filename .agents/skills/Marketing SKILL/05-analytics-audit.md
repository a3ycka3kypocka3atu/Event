# Аналітика, аудит кампаній та прогнозування — Повний довідник

Цей документ — повна база знань з аналітичної інфраструктури, аудиту рекламних кампаній, побудови дашбордів та прогнозування. Використовуй його коли потрібно провести аудит кампаній, знайти bottleneck, побудувати дашборд або спрогнозувати результати.

---

## 1. Технічна інфраструктура аналітики

### 1.1. Google Analytics 4 (GA4)

**Обмеження:**
- Трекінг на cookies + Client ID. Очистка куків = новий юзер. Точний LTV >30 днів неможливий
- Семплювання при >10 млн івентів (обрізання даних)
- Поріг конфіденційності: <30 юзерів у розрізі = зникають зі звіту
- Затримка даних: 8-48 годин

**Важливі звіти:**
- **Demographics:** Вік, стать. Аналізуй частку "Unknown" — може бути >50%, що робить висновки хибними
- **Acquisition:** Джерела трафіку. Дефолтне групування працює погано (багато "Unassigned"/"Direct"). Обов'язково робити Custom Channel Grouping на основі UTM-міток
- **Monetization:** Дохід, середній чек, ефективність промо-акцій
- **Predictive Audiences:** AI виділяє "Likely 7-day purchasers", "28-day top spenders", "Likely 7-day churning users". Потрібно мінімум 1000 юзерів з покупкою і 1000 без за 28 днів
- **Атрибуція:** Data-Driven Attribution (DDA) за замовчуванням. Розподіляє цінність між усіма каналами (справедливіше, ніж Facebook, який присвоює 100% собі)

### 1.2. Google Tag Manager (Server-Side)

**Навіщо:** Звичайний GTM працює у браузері → блокується AdBlock-ерами. Server-Side GTM — окремий контейнер на хмарному сервері.

**Як працює:** Дані з браузера → один запит на сервер → сервер розподіляє в GA4, Meta, TikTok. Користувач не знає куди далі йдуть дані.

**Переваги:**
- Обхід блокувальників реклами
- Пришвидшення завантаження сайту (менше скриптів у браузері)
- Збагачення даних: додавання email/телефону з CRM до сесії неавторизованого юзера → відправка в рекламні платформи

### 1.3. Conversion API (CAPI) / Enhanced Conversions

**Meta CAPI:** Прямий зв'язок сервер → Meta. Обходить iOS 14.5+ та блокування cookies. Працює разом із Pixel для максимізації даних (Event Match Quality). Тести: +19-39% більше трекінгу покупок.

**Методи впровадження:**
- CAPI Gateway (через AWS — найшвидше)
- Partner Integration (Shopify, WooCommerce)
- Direct Integration (програміст, 2-4 тижні)

**Google Enhanced Conversions:** Відправка хешованих email/телефонів (навіть офлайн-покупки) для ідентифікації юзера без cookies.

### 1.4. MMP (Mobile Measurement Partners)

**AppsFlyer, Adjust, Singular** — атрибуція для мобільних додатків (замість Pixel використовується SDK).

**Можливості:** Трекінг платного та органічного трафіку, мультиканальний репортинг, когорти для LTV, SKAdNetwork, діплінки (Web2App).

### 1.5. SKAdNetwork (SKAN)

**Як працює:** Фреймворк Apple для iOS без IDFA. Дані агреговані, із затримкою 24-48 годин. При малому обсязі інсталів — Privacy Threshold (дані приховуються, Null Conversion Value).

**Обмеження:** 64 слоти (6 біт) для кодування конверсій. Три режими:
- Revenue Mode (діапазони доходу)
- Conversion Mode (факт події)
- Engagement Mode

**SKAN 4.0:** Три вікна конверсій (до 2 днів, 3-7 днів, 8-35 днів). Coarse Grained Values (низька/середня/висока).

## 2. Дашборди та візуалізація

### 2.1. Автоматизований пайплайн

```
AppsFlyer Data Locker (безкоштовно)
    ↓ (щоранку автоматично)
Google BigQuery (сирі дані, SQL-запити)
    ↓
Looker Studio (візуалізація, графіки, фільтри)
```

**Вимоги:** Безкоштовний ліміт BigQuery — 1 ТБ/міс. Потрібен SQL для написання запитів.

### 2.2. Метрики на дашборді

**Базові:** Spend, Installs/Units, CPI, CPM, CTR, CVR

**Фінансові:** Revenue, ROAS, ARPU, AOV

**Порівняльні (обов'язкові):** Relative Change % — відносна зміна до попереднього тижня/періоду ("Spend +20%", "ROAS -5%")

**Для креативів:** Share of Spend (% бюджету на креатив)

### 2.3. Типи дашбордів

**Campaign Report (напівавтоматичний Google Sheets):**
- 3 CSV з MMP (7 днів, 30 днів, вчора)
- VLOOKUP зводить у таблицю
- Структура: Campaign ID | Metrics This Week | Metrics Last Week | % Change
- Колонка для коментарів і рішень по бюджету

**Creative Report:**
- Назва креативу | % Спенду | CPI | Conversion Rate | Retention Rate
- За 5 хвилин показує клієнту який креатив виграє

**Looker Studio Dashboard:**
- Візуальні графіки трендів (Spend та Installs по днях)
- Фільтри по країнах
- Теплові карти (Heatmaps)
- Умовне форматування (зелений/червоний при відхиленні KPI)

## 3. Аудит рекламних кампаній

### 3.1. Принцип аудиту

Не використовуй сліпі шаблони. Аналіз йде по ланцюжку: Закупка (кабінет) → Воронка (продукт) → Інфраструктура (трекінг/оплати).

### 3.2. Алгоритм аудиту

**Крок 1 — Перевірка закупки (рекламний кабінет):**
- CPM виріс? → Трафік став дорожчим (сезонність, аукціон)
- Frequency зросла при падінні CTR? → Вигорання креативу/аудиторії
- CTR високий (10%), але конверсій нуль? → Клікбейт або фрод
- Брейкдауни: перевір країни, девайси, плейсменти. Бюджет може зливатись на Audience Network або десктоп

**Крок 2 — Перевірка воронки (AARRR або STDC):**
- Де відвал? Приклад: інстал → реєстрація 40% (норма), реєстрація → покупка 0.1% (проблема)
- Message Match: креатив обіцяє одне, лендінг показує інше?
- Ціна: може бути зависока для холодного трафіку?

**Крок 3 — Перевірка інфраструктури:**
- Працює платіжний шлюз? Не відвалився PayPal?
- Кнопки працюють у In-App браузері Facebook/Instagram?
- Трекінг коректний? Розходження між базою та кабінетом >15%?

### 3.3. Діагностика: що саме зламалось?

| Симптом | Діагноз | Дія |
|---------|---------|-----|
| Високий CPM, низький CTR, висока Frequency | Вигорання креативу | Нові креативи, Variation/Refresh |
| Високий CTR, але 0 конверсій | Клікбейт або фрод, або зламаний лендінг | Перевір Message Match, тестуй лендінг |
| CTR ок, трафік іде, покупок нема | Проблема у воронці/ціні/оплатах | CRO-аудит лендінгу, перевір checkout |
| Низький CPM, але нема результату | "Сміттєвий" дешевий трафік | Перевір плейсменти, виключи Audience Network |
| CPA росте через 2-3 дні | Аудиторія занадто вузька | Розширити таргетинг, Broad |
| Розходження даних кабінет vs база >20% | Проблема з трекінгом | Перевір CAPI, Pixel, Server-Side GTM |

### 3.4. Фреймворк Bottleneck-аналізу

Побудуй воронку метрик і знайди де найбільше падіння:

```
Impressions (показів)
    ↓ CTR (клікабельність)
Clicks (кліків)
    ↓ Landing Page View Rate
Landing Page Views
    ↓ CVR (конверсія сайту)
Add to Cart / Lead
    ↓ Checkout Rate
Purchases / Sales
    ↓ Upsell Rate
Revenue
```

Де найбільше падіння у % — там bottleneck. Фокусуй оптимізацію на цьому етапі.

## 4. Data Science і прогнозування

### 4.1. Meta Prophet — Прогнозування CPM, CPA, ROAS

**Що це:** Відкрита Python/R бібліотека від Meta для прогнозування часових рядів (Time-Series).

**Як працює:** Будує прогнозні коридори (Lower/Upper). Розуміє річну сезонність та свята.

**Точність:** На реальних проєктах похибка прогнозу CPM — 5-10%.

**Приклади використання:**
- Прогноз CPM: знаючи що через місяць CPM зросте (високий сезон) — заздалегідь готуєш бюджет або креативи
- Прогноз CPA з регресором Spend: "Що буде з CPA якщо збільшимо бюджет на 20%?"
- Прогноз ROAS двома шляхами:
  - Прямий: прогнозуєш ROAS як метрику
  - Розрахунковий: прогнозуєш CPM, CTR, CVR, AOV окремо → вираховуєш ROAS за формулою

**Вимоги до даних:**
- Time-Series з деталізацією по днях (не тижнях)
- Мінімум 1 рік історії (для сезонності), ідеально 2-3 роки
- Однорідність: якщо AOV стрибає $20-$100 щодня → прогноз хибний. Коридор $30-$50 → похибка ~5%

**MAPE (оцінка точності):** <10% — супер. 10-20% — добре. >50% — модель хибна.

### 4.2. ChatGPT (Advanced Data Analysis)

Завантажуєш CSV без знання Python:
- Очистка даних (заміна missing values)
- Пошук кореляцій
- Статистичний аналіз, кластеризація
- Створення графіків (Heatmaps)

**Приклад промпту:** "Відфільтруй дані, де ROAS > 24% та покупок > 20. Видай список у розрізі Campaign ID та Країна" → готова таблиця топ-зв'язок для масштабування.

**Застереження:** Не завантажуй конфіденційні дані (імена, назви). Використовуй деанонімізовані ID або Enterprise версію.

### 4.3. Probabilistic моделювання

Передбачення конверсій без IDFA. MMP (Adjust) використовує патерни поведінки, IP, пристрої (Fingerprinting). Впровадження Adjust у серпні 2023 → Day-0 ROAS для iOS ×3. Meta також використовує подібний підхід (AEM).

## 5. Когортний аналіз

### 5.1. Як проводити

Групуєш юзерів за датою залучення (когорта "Вересень 2023"). Відслідковуєш дохід на 1-й, 2-й, 6-й, 12-й місяць.

### 5.2. Що витягнути з когорт

**Ребіли:** Скільки % поновлюють підписку? (типово ~20% для річної). Цей коефіцієнт → у фінмодель.

**Refund Rate:** Історичний % повернень (~10-11%).

**Payback Period:** За скільки місяців когорта повертає 100% CAC? Якщо Return <100% за 6 місяців → масштабувати НЕ МОЖНА.

**LTV-крива:** Як росте кумулятивний дохід з часом. Плато = стеля LTV.

### 5.3. Шаблон когортної таблиці

```
Когорта    | M0    | M1    | M3    | M6    | M12   | LTV
-----------|-------|-------|-------|-------|-------|------
Січень     | $5.00 | $6.50 | $8.20 | $10.0 | $12.5 | ...
Лютий      | $4.80 | $6.20 | $7.90 | ...   | ...   | ...
Березень   | $5.20 | $6.80 | ...   | ...   | ...   | ...

M0 = Initial ARPU (дохід у день покупки)
M1 = Кумулятивний дохід за 1 місяць
...і так далі
```

## 6. Аудит-чекліст для AI-агента

Використовуй цей алгоритм коли тобі скидають дані кампаній для аналізу:

```
АУДИТ РЕКЛАМНОЇ КАМПАНІЇ

1. ЗБІР ДАНИХ
   □ Які метрики надані? (Spend, CPI, CTR, CVR, ROAS, Revenue)
   □ За який період?
   □ Є розбивка по кампаніях/країнах/креативах?
   □ Є порівняння з попереднім періодом?

2. ПЕРЕВІРКА ЗДОРОВ'Я МЕТРИК
   □ CPM — в нормі для ніші/гео? Зріс чи впав?
   □ CTR — >1% для відео? Аномально високий (клікбейт)?
   □ CVR — в нормі для типу воронки? (E-com ~3%, Lead Gen ~4-6%)
   □ Frequency — <3? Якщо вище = вигорання
   □ ROAS — вище Break-even? (Break-even = 1/Маржа)

3. BOTTLENECK-АНАЛІЗ
   □ Де найбільше падіння у воронці?
   □ Impressions → CTR: проблема креативу
   □ CTR → CVR: проблема Message Match або лендінгу
   □ CVR → Purchase: проблема ціни/оплат/checkout

4. РОЗРІЗИ (BREAKDOWNS)
   □ По країнах: яке гео найприбутковіше?
   □ По плейсментах: де зливається бюджет?
   □ По креативах: який % бюджету забирає кожен?
   □ По девайсах: Mobile vs Desktop

5. РЕКОМЕНДАЦІЇ
   Для кожного bottleneck:
   □ Що конкретно зробити
   □ Який очікуваний ефект
   □ Пріоритет (High/Medium/Low)
   □ Зв'язок з іншими модулями (потрібні нові креативи? CRO?)
```

## 7. Бенчмарки аналітики

| Метрика | Норма | Проблема |
|---------|-------|----------|
| Розходження кабінет vs база | 5-10% (до 15%) | >20-30% = проблема трекінгу |
| Refund Rate | ~10-11% | >15% = проблема продукту/очікувань |
| Frequency (Meta/TikTok) | 1.5-3.0 | >3.0 = вигорання аудиторії |
| MAPE прогнозу | <10% супер, 10-20% добре | >50% = модель хибна |
| Day-0 ROAS з Probabilistic | ×3 від детерміністичного | Adjust/Meta AEM |

## 8. Зв'язки з іншими модулями

- **Аналітика → Юніт-економіка:** Когортний аналіз дає реальний LTV, Payback Period, Refund Rate → все йде у фінмодель
- **Аналітика → Креативи:** Дашборд показує Share of Spend, CTR, CVR на креатив → рішення про Variation/Refresh або вимикання
- **Аналітика → Воронки:** Bottleneck-аналіз показує де відвалюється трафік → CRO оптимізація конкретного етапу
- **Аналітика → Масштабування:** True ROAS > Break-even ROAS + стабільність 2-4 тижні = зелене світло на скейл
- **Трекінг → Алгоритми платформ:** Без CAPI/Server-Side GTM автоматизовані кампанії (Advantage+, PMax, SPC) сліпі — не можуть знайти платоспроможну аудиторію через обмеження iOS 14+
