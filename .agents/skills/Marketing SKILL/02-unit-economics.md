# Юніт-економіка та фінансове моделювання — Повний довідник

Цей документ — повна база знань з розрахунку юніт-економіки, прайсингу та фінансового моделювання маркетингових кампаній. Використовуй його коли потрібно порахувати окупність, визначити допустимий CAC, побудувати фінмодель або прийняти рішення про масштабування.

---

## 1. Ключові метрики

### 1.1. LTV (Lifetime Value — Життєва цінність клієнта)

Загальна сума доходу від клієнта до моменту відвалу (churn).

**Формула:** `LTV = Contribution Margin / Customers`

**Періоди розрахунку:** 30, 60, 180, 365 днів, 5 або 10 років. Повноцінний LTV видно через когортний аналіз.

**True LTV vs Initial ARPU:** Initial ARPU — дохід у день покупки. True LTV — сукупний дохід на дистанції (включає ребіли, апсели, повторні покупки). Різниця може бути в 2-5 разів.

### 1.2. CAC (Customer Acquisition Cost)

**Формула:** `CAC = Витрати на маркетинг / Кількість залучених клієнтів`

**Важливо розділяти:**
- Fixed costs (фіксовані): розробка, зарплати — не масштабуються з трафіком
- Variable costs (змінні): реклама, агентство — напряму залежать від обсягу

**Золоте співвідношення:** CAC : LTV = 1 : 3. При LTV $1000 → допустимий CAC ~$300.

### 1.3. ROAS (Return on Ad Spend)

**Базова формула:** `ROAS = Revenue / Ad Spend`

**True ROAS:** Враховує не лише першу покупку, а й прогнозований LTV. Приклад: ROAS з першої покупки = 1.0, True ROAS за рік = 1.2.

**Break-even ROAS:** Мінімальний ROAS для виходу в нуль.
```
Break-even ROAS = 1 / Маржа

Приклад: маржа 55% (0.55) → Break-even ROAS = 1.81
```

**Математичний розрахунок ROAS:**
```
ROAS = AOV / ((CPM / 1000 / CTR) / CVR)

Де:
  AOV = середній чек
  CPM = вартість 1000 показів
  CTR = клікабельність
  CVR = конверсія сайту
```

### 1.4. ARPU / ARPPU

- **ARPU** — середній дохід на одного користувача (включаючи тих, хто не платить)
- **ARPPU** — середній дохід на платящого користувача
- **Initial ARPU** — дохід у день покупки (без ребілів і апселів)

### 1.5. Payback Period (Період окупності)

Час, за який LTV перекриває CAC.

**Нормальні значення:**
- E-commerce: 0-3 місяці (ідеально окупність з першої покупки)
- SaaS/Subscription: 6-12 місяців
- Health & Fitness (великі компанії): до 24 місяців (при наявності капіталу)
- Інфобізнес: 1-6 місяців

**Правило:** Бізнес далеко не завжди окупається в перший місяць. Payback Period 6-12 місяців — нормально, якщо є капітал для фінансування росту.

## 2. Побудова фінансової моделі

### 2.1. Вхідні дані для калькулятора

Три типи калькуляторів під різні вертикалі:

**E-commerce калькулятор:**
```
Вхідні дані:
  CPC (вартість кліку): ___
  Місячний бюджет: ___
  CVR (конверсія сайту): ___
  AOV (середній чек): ___
  Management fee (оплата агентству): ___
  Маржинальність: ___
  
Автоматичний розрахунок:
  Кліки = Бюджет / CPC
  Продажі = Кліки × CVR
  Revenue = Продажі × AOV
  CAC = Бюджет / Продажі
  Break-even ROAS = 1 / Маржа
  Profit = (Revenue × Маржа) - Бюджет - Fee
```

**Lead Gen калькулятор:**
```
Вхідні дані:
  CPL (вартість ліда): ___
  Конверсія лід → дзвінок: ___
  Конверсія дзвінок → продаж: ___
  Чек продукту: ___
  
Автоматичний розрахунок:
  CAC = CPL / (Conv_call × Conv_sale)
  Revenue per lead = Чек × Conv_call × Conv_sale
  ROAS = Revenue / Spend
```

**Tech/Інфобізнес калькулятор:**
```
Вхідні дані:
  CPI (вартість інсталу) або CPL: ___
  Конверсія в підписку: ___
  Ціна підписки: ___
  Коефіцієнт ребілів: ___
  Refund Rate: ___
  Комісія стору: ___
```

### 2.2. Прогнозування ребілів (поновлення підписки)

Використовується когортний аналіз:
1. Візьми когорту (напр., хто купив річну підписку в січні)
2. Подивись скільки поновили через рік
3. Цей коефіцієнт (напр., 0.20 = 20%) закладай у фінмодель

**Приклад:**
```
Когорта: 1000 річних підписок × $50 = $50,000
Ребіл 20%: 200 × $50 = $10,000
LTV когорти за 2 роки: $60,000
LTV на юзера: $60
```

### 2.3. Моделювання без власних даних

Якщо стартап без даних — їх НЕ можна придумати. Варіанти:

1. **Data.ai / SensorTower:** Середній Revenue per Install у ніші
2. **Бенчмарки конкурентів:** Скопіюй прайсинг лідера (стратегія Follower)
3. **Ком'юніті:** Запитай в партнерів/конкурентів/експертів
4. **Google Keyword Planner:** CPC в ніші → моделюй CAC при конверсії 2-3%
5. **Тестовий бюджет:** Заклади 5-10× від цільового CPA на перший тест, збери реальні дані

## 3. Бенчмарки ринку

### 3.1. Конверсії

| Метрика | Бенчмарк | Контекст |
|---------|----------|----------|
| CVR E-commerce сайт | ~2.86% (~3%) | Shopify, глобально |
| CVR лендінг → лід | ~6% | Lead Gen |
| Лід → призначений дзвінок | ~20% (кожен 5-й) | Lead Gen |
| Дзвінок → продаж | 15-30% | Lead Gen |
| Інстал → реєстрація (App) | 40-70% (відвал 30-60%) | Mobile Apps |
| Інстал → покупка (App) | ~2% | Mobile Apps |
| Вебінар → покупка (High-Ticket, cold) | ~0.6% | Кейс Лиманюка |
| Реєстрація → оплата (High-Ticket) | ~10% | Кейс Лиманюка |

### 3.2. Юніт-економіка

| Метрика | Бенчмарк | Контекст |
|---------|----------|----------|
| CAC : LTV | 1 : 3 | Золоте співвідношення |
| Refund Rate (Apps) | ~11% | Повернення коштів |
| Комісія App Store / Google Play | 30% (15% для малих) | + податки = ~33-34% |
| Ребіл (річна підписка) | ~20% | Поновлення на наступний рік |
| Payback Period (норма) | 6-12 місяців | Subscription бізнес |

## 4. Прайсинг

### 4.1. Вибір моделі монетизації

| Модель | Коли обирати | Вплив на юніт-економіку |
|--------|-------------|------------------------|
| **Freemium** | Великий TAM, віральний продукт | Великий CAC (мільйони юзерів), дохід з малого % |
| **Trial (3-7 днів)** | SaaS, Apps з прив'язкою картки | Ключовий KPI: Trial-to-Paid rate |
| **Subscription** | Повторюване використання | Фокус на Retention, LTV росте з часом |
| **One-time purchase** | E-com, інфопродукти | Фокус на AOV і повторних покупках |
| **Marketplace** | Двосторонній ринок | Комісія від транзакцій, потрібні обидві сторони |
| **Рекламна / Offer Wall** | Величезні об'єми трафіку | Низький ARPU, потрібні мільйони MAU |

### 4.2. Визначення ціни

**Якщо немає даних (стратегія Follower):**
- Не став ціну вищу за конкурентів
- Скопіюй прайсинг і пейволи лідерів — вони вже перевірені ринком

**Якщо є дані (оптимізація):**
- A/B тест цін (підвищення на 10-20% і вимірювання впливу на конверсію)
- Pricing Ladder: дешевший → основний → преміум

### 4.3. Pricing Anchor (Психологія ціноутворення)

**3 тарифи:** Завжди показуй 3 варіанти. Виділяй середній або річний як "Best Value" / "Найпопулярніший".

**Розбивка ціни:** Великим шрифтом: "лише $0.54/день". Маленьким: "списується $26 за рік".

**Закреслена ціна:** Показуй стару ціну закресленою і суму економії ("Ви економите $19.90").

**Ефект приманки (Decoy):** Додай третій варіант, який робить цільовий тариф вигідним у порівнянні. Приклад:
```
Місячний: $12.99/міс
Річний: $4.99/міс (списується $59.99) ← BEST VALUE
Тижневий: $4.99/тиждень ($260/рік) ← Decoy, робить річний вигідним
```

### 4.4. Вплив прайсингу на воронку

| Ціна продукту | Тип воронки | CAC ліміт |
|---------------|-------------|-----------|
| $5-50 | Quiz, Advertorial, Tripwire | $5-15 |
| $50-300 | VSL, вебінар, лід-магніт | $15-80 |
| $300-1000 | Діагностичний дзвінок, прогрів | $80-300 |
| $1000+ | Ecosystem (YouTube, Telegram, дзвінок) | $150-500+ |

Продавати $900 продукт "в лоб" = конверсія 0.6%. Високий чек вимагає зміни воронки.

## 5. Оптимізація юніт-економіки

### 5.1. Головний принцип

**Малий бізнес:** 80% зусиль на зниження CAC (вимагають дешевих лідів).
**Великі гравці:** 80% зусиль на збільшення LTV, 20% на CAC.

**Правило:** Той, хто може платити за клієнта найбільше — виграє ринок. Не знижуй CAC — збільшуй LTV.

### 5.2. Важелі збільшення LTV

**AOV (середній чек):**
- Бандли ("Купи 2 зі знижкою")
- Order Bump (чекбокс на чекауті)
- Post-purchase One-Click Upsell
- Welcome Gift для створення надцінності (AG1: шейкер безкоштовно)
- Cross-sell (супутні товари)
- Підвищення ціни з A/B тестом

**Retention (утримання):**
- Email/SMS-ланцюжки
- Ком'юніті (закритий клуб, Telegram)
- Push-сповіщення (Apps)
- Якісний клієнт-сервіс (Care у STDC)
- Програми лояльності
- Double Subscription (AG1: відразу 2 пачки)

**Repeat Purchase (повторні покупки):**
- Підписна модель замість одноразової
- Автоматичне поповнення (Subscription + auto-refill)
- Ексклюзивний контент для існуючих клієнтів

### 5.3. Коли масштабувати — Чекліст

```
✅ True ROAS > Break-even ROAS
✅ CAC : LTV = 1:3 або краще
✅ Payback Period < 12 місяців (або є капітал покрити)
✅ Є мінімум 100+ конверсій для статистичної значущості
✅ Юніт-економіка стабільна 2-4 тижні поспіль

❌ НЕ масштабуй якщо:
  • CAC > LTV (збільшення бюджету = швидше банкрутство)
  • Дані базуються на <50 конверсіях
  • ROAS нестабільний (скаче день до дня)
  • Немає моделі LTV на дистанції
```

## 6. Кейси з числами

### Кейс 1: E-commerce — Від мінуса до плюса через CRO і AOV

```
БУЛО:
  Бюджет: $5000, CPC: $1 (5000 кліків)
  CVR: 1%, AOV: $100, Маржа: 55%, Fee: $1000
  Продажі: 50, Revenue: $5000
  CAC: $100
  Profit: -$3,250 ❌

СТАЛО (після CRO + апселів):
  CVR: 2% (покращення лендінгу)
  AOV: $125 (додані бандли і апселі)
  Продажі: 100, Revenue: $12,500
  CAC: $50
  Profit: +$875 ✅

Зміна: CVR ×2 + AOV +25% = з мінуса в плюс без зниження CPC
```

### Кейс 2: Влад Лиманюк — Зламана юніт-економіка High-Ticket

```
Бюджет: $5,000
Кліки: 10,000, Ліди: 3,500 (CPL: $1.50)
Глядачі вебінару: 712 (доходимість 20%)
Продажі: 4 × $900 = $3,700
Конверсія глядач → покупка: 0.6%
CAC: $1,250

Profit: -$1,300 ❌
CAC:LTV = 1:0.72 (має бути 1:3)

Рішення: Змінити воронку на діагностичний дзвінок
  Варіант A: Продукт $600, цільовий CAC $150
  Варіант B: Продукт $1000, цільовий CAC $300
```

### Кейс 3: Мобільний додаток — Кредитна лінія і банкрутство

```
Скейлінг: з $100K до $1.5M/міс
Аудит показав відсутність фінмоделі

Розрахунок когорти:
  Річні підписки: X шт
  - Refund Rate: 11%
  - Комісія стору + податки: 33-34%
  - Ребіл (поновлення): 20%
  
Результат: когорта НЕ окупається за 6 місяців
Рішення: зупинити кредитну лінію, зрізати спенди
```

### Кейс 4: AG1 — Фокус на LTV = $600M/рік

```
1 флагманський продукт (без розпорошення)
Модель: тільки підписка (не одноразово)
Трюки:
  - Double Subscription (відразу 2 пачки)
  - Welcome Kit (шейкер + ложка безкоштовно)
  - Надцінність: сприйнята цінність > ціна
  
Результат: $600M доходу у 2024 році
```

## 7. Формули — Шпаргалка

```
CAC = Ad Spend / Customers
LTV = Total Revenue from Customer over Lifetime
ROAS = Revenue / Ad Spend
Break-even ROAS = 1 / Margin
ROAS (розрахунковий) = AOV / ((CPM / 1000 / CTR) / CVR)
CPA = CPC / CVR
CPL = Ad Spend / Leads
Profit = (Revenue × Margin) - Ad Spend - Fixed Costs
Payback Period = CAC / Monthly Revenue per Customer
Ребіл-прогноз = Когорта × Коефіцієнт ребілу
```

## 8. Зв'язки з іншими модулями

- **Юніт-економіка → Вибір воронки:** CAC і ціна продукту диктують тип воронки. $30 товар = Advertorial з апселами. $900 послуга = прогрів + діагностичний дзвінок.
- **Юніт-економіка → Креативи:** Бюджет на тест = 5-10× від CPA. Креатив з поганим CTR, але хорошим CPA — НЕ вбивати.
- **Юніт-економіка → Масштабування:** True ROAS > Break-even ROAS = зелене світло. Інакше масштабування = прискорення банкрутства.
- **CRO → Юніт-економіка:** Подвоєння CVR з 1% до 2% = подвоєння продажів без додаткового бюджету = зниження CAC вдвічі.
- **Ресерч ринку → Юніт-економіка:** CPC з Keyword Planner + CVR бенчмарк = модель CAC до запуску.
